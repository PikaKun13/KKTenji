# KKTenji v1 実装計画

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 設計書 v1 の MVP — sample deck を開いて思維関係図を描画・操作・プレゼンでき、NSIS インストーラまで作れる Electron アプリ。

**Architecture:** core（純ロジック・Vitest 対象）/ ui（SVG+DOM 直描画、承認済みモックアップ `mockup/kktenji-mockup.html` からの移植が正）/ shell 抽象層（browser 開発用 + electron 実装）。設計書 = `docs/superpowers/specs/2026-07-19-kktenji-design.md`。

**Tech Stack:** TypeScript + Vite + Vitest / Electron + electron-builder(NSIS) / marked + DOMPurify（md 描画）/ PowerShell COM（pptx→PNG）。

## Global Constraints

- UI 文言は日本語。ドキュメントも日本語 + 英語術語
- sidecar は不可信入力: title/summary/label/notes は textContent 描画、md は DOMPurify で sanitize
- モーション数値は `src/ui/motion.ts` の token のみ（設計書 §6.5 の値。直書き禁止）
- shell API 直呼び禁止（`src/shell/api.ts` の `ShellApi` 経由）
- 「寛容パース・大声報告」: 設計書 §4.7 の表に従う
- コミットは意味単位、末尾に `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`
- テスト報告は実数（passed/failed/skipped）

### Task 1: core/types

**Files:** Create `src/core/types.ts`

**Produces（後続タスク全部がこれに依存）:**

```ts
export type LinkType = 'support' | 'echo' | 'contrast' | 'cause';
export interface TenjiNode { id: string; title: string; parent: string | null;
  page: number | null; summary?: string; notes?: string; anchor?: string | null; contentHash?: string; }
export interface TenjiLink { from: string; to: string; type: LinkType;
  direction: '->' | '<->'; label?: string; }
export interface TenjiSource { type: 'pptx' | 'md'; path: string; pageBy?: 'h2' | 'hr'; }
export interface TenjiDoc { version: number; deckId?: string; title: string; source: TenjiSource;
  nodes: TenjiNode[]; links: TenjiLink[]; flow?: string[]; [k: string]: unknown; }
export interface Diagnostic { level: 'error' | 'warn'; code: string; message: string; nodeId?: string; }
export interface ParsedDeck { doc: TenjiDoc; nodes: Map<string, TenjiNode>;
  children: Map<string, TenjiNode[]>; roots: TenjiNode[]; links: TenjiLink[]; diagnostics: Diagnostic[]; }
```

- [ ] 型定義のみ。Commit `feat(core): tenji.json v1 型定義`

### Task 2: core/parser — 寛容パース・大声報告

**Files:** Create `src/core/parser.ts`, Test `src/core/parser.test.ts`

**Produces:** `parseTenji(jsonText: string): { deck?: ParsedDeck; fatal?: Diagnostic }`

処理順: JSON.parse(失敗=fatal) → 必須形状検査(nodes 配列なし=fatal) → id 重複(後者無効+warn `dup-id`) →
parent 不在(ルート化+warn `orphan`) → 親子循環(検出辺切断+warn `cycle`) → link 端点不在/自環(捨てる+warn
`bad-link`/`self-loop`) → 完全重複 link 排除(warn `dup-link`) → 逆向きペア `->`×2 を `<->` 1 本に正規化
(warn `two-way-merge`) → 未知 link type(そのまま保持+warn `unknown-type`) → flow 不明 id(除去+warn
`bad-flow-ref`) → 0 ノード(deck は返す+warn `empty`)。

- [ ] テストを先に書く（上記 10 ケース + 正常系で roots/children が正しい）
- [ ] 実装 → 全テスト green
- [ ] Commit `feat(core): 寛容パーサ+診断 (設計書§4.7)`

### Task 4: core/layout — 決定的 tidy tree

**Files:** Create `src/core/layout.ts`, Test `src/core/layout.test.ts`

**Produces:**

```ts
export interface NodeBox { x: number; y: number; w: number; h: number; depth: number; }
export interface LayoutResult { boxes: Map<string, NodeBox>; bounds: { x: number; y: number; w: number; h: number }; }
export function layoutDeck(deck: ParsedDeck): LayoutResult;
```

モックアップの `layout()` を一般化: 深さ d のサイズ = SIZE[min(d,2)]（250×92 / 222×74 / 200×58）、
列 X = COLX[min(d,2)]（70 / 440 / 830）+ d>2 は 830+(d-2)*390。葉を y 累積（GAP 26）、親は子範囲の中央。
複数ルートは縦に連結（ルート間 GAP 52）。

- [ ] テスト: 同一入力 2 回で同一結果（決定性）/ 親は子の中央 / 森で重ならない / bounds が全 box を包含
- [ ] 実装 → green → Commit `feat(core): 決定的レイアウト`
